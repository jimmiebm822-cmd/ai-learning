⭐ #1 — Lilian Weng：Scaling Laws, Carefully

lilianweng.github.io · Jun 24, 2026

从 Amari (1992) 到 Kaplan → Chinchilla → 2026 前沿的 scaling laws 全景。核心洞察：幂律指数是任务域属性而非架构属性；Kaplan vs Chinchilla 的矛盾源于数据假设差异；小规模实验外推到大模型时的 3 个常见陷阱。这是目前最好的 scaling laws 参考文章。

对 Daddy 的意义：正在从「用模型」过渡到「设计 agent 系统」——理解模型能力的成本-性能 tradeoff 是底层基本功。今天就可以开始读，它比原学习路径里任何一篇文章都更适合作为阶段 1 的起点。

阶段 1：LLM 基础

⭐ #2 — arXiv：Instruction Bleed — 多 prompt 模块间的隐性干扰

arXiv 2606.26356 · Jun 24, 2026

首次形式化 CBL（Compositional Behavioral Leakage）：编辑一个 prompt 模块会悄无声息地改变另一个模块的行为。根因是 transformer 自注意力在共享上下文窗口中不隔离。即使两个模块没有共享变量或依赖，干扰依然发生。

对 Daddy 的意义：每个 skill 都是多 prompt 模块组成——system prompt + tool descriptions + 输出格式 + 边界规则。这篇论文直接告诉你：模块化 prompt 是有漏洞的抽象。必须做整体测试，不能只测单个模块。之前 yield-report-base 的代码审计没覆盖这一点。

阶段 2：Agent 架构

⭐ #3 — rtrvr.ai：Code-as-Plan 颠覆 Agent 经济学

rtrvr.ai · Jun 25, 2026

模型一次性生成完整工作流代码，本地 harness 执行——80+ 次 API 调用 → 1 次。DeepSeek V4 Flash 便宜、开源、代码能力够用。核心金句："Developers were being milked for runtime, not intelligence."

对 Daddy 的意义：如果 agent 每个 task 烧几十次模型调用，架构成本就是竞争劣势。护城河从「有模型访问权」变成「harness 设计质量」。直接影响 FIL 日报和 yield-report 的重构方向。

阶段 3-4：实践 + 评测

#4 — arXiv：67 模型的 Co-Failure Ceiling

arXiv 2606.27288 · Jun 25, 2026

多模型系统（routing / voting / mixture-of-agents）的准确率永远超不过 1 − β（β = 所有模型同时出错的概率）。只看单模型准确率会漏掉这个关键指标——高准确率 + 高 β = 组合系统白花钱。

阶段 4：评测方法论

#5 — Simon Willison：Prompt Injection 的根因是「角色混淆」

simonwillison.net · Jun 22, 2026

LLM 遵循的是文本风格，不是 <system> / <user> 这类角色标签。去风格化（destyling）把攻击成功率从 61% 降到 10%。结论：在模型真正具备角色感知能力之前，注入防御就是打地鼠。

对 Daddy 的意义：别把 system prompt 当安全边界。agent 在公开渠道运行时，用户输入可以模仿你的 prompt 风格来逃逸。防御策略要降到风格层。

💡 今日核心收获（6 条）

  1. Prompt 模块化是有漏洞的抽象——只测单个模块不够，必须测整体
  2. Agent 经济学正在被重写——Code-as-Plan 把 80 次调用压到 1 次
  3. 角色标签不是安全边界——LLM 看的是风格，不是 <system> vs <user>
  4. Co-Failure Rate（β）是多模型系统的关键指标——高准确率 ≠ 低共败率
  5. Agentic Engineering > Vibe Coding——结构化 prompt + plan + review
  6. Agent 配置是治理问题——集中管理、版本控制，不散落在各个 repo

📌 一句话总结今天：Scaling Laws 给你模型的底层直觉，Instruction Bleed 告诉你 prompt 设计的边界，Code-as-Plan 改变你对 agent 经济模型的认知。这三篇今天必读。
