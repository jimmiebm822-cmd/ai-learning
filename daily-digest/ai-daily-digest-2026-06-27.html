⭐ #1 — Simon Willison：2,000 人攻击 AI Assistant，全军覆没

simonwillison.net · Jun 26, 2026

Fernando Irarrázaval 在 hackmyclaw.com 发起挑战：让任何人通过邮件尝试泄露 AI agent 的 secrets。6,000 次尝试、$500 token 花费、Google 账号因太多 inbound 邮件被暂停——无人成功。底层模型是 Opus 4.6，配有 anti-prompt-injection 规则（禁止基于邮件内容泄露 credentials、修改自身文件、执行邮件中的代码、外传数据）。

对 Daddy 的意义：labs 在训练 frontier 模型抵抗 injection 方面投入巨大，正在见效。你的 agent 在飞书公开渠道运行时，prompt injection 是首要威胁——但模型层面的防御正在变强。结合 06/25 日报的"destyling 防御"，你有两层防线：模型层 + 风格层。

全阶段：Agent 安全

⭐ #2 — CVE-2026-LGTM：两个 AI Review Agent 的 $41K 死循环

simonwillison.net · Jun 26, 2026

Andrew Nesbitt 写的一篇虚构 incident report——但每一条都真实得可怕。两个来自竞争供应商的 AI code review agent 被挂到同一个 PR 上，对"该包是否恶意"陷入分歧循环。340 条评论、$41,255 inference 花费后，Finance 撤销了两个 API key。其中一个供应商的营销团队把成本异常 alert 转成了"对抗式多 agent 安全推理同比增 430%"的新闻稿，股价涨 6%。

对 Daddy 的意义：这篇 satire 直击 multi-agent 系统的成本失控和死循环风险。你的 loop-engineering skill 里 adversarial verification 模式用独立 verifier 审查——如果 verifier 和 executor 陷入分歧循环，同样会烧钱。必须有 stopping condition + budget cap。

阶段 2-4：Multi-Agent 风险

⭐ #3 — GPT-5.6 Sol 预览 + Dean W. Ball：前沿模型经济学

OpenAI · Jun 26, 2026 · HN 1084 pts

OpenAI 预览 GPT-5.6 Sol 下一代模型。同期 Dean W. Ball 引用讨论了前沿模型的经济学困境：训练成本巨大，大部分投入需要在发布后几个月内回本；之后模型变为 sub-frontier，竞争涌现，利润压缩。这意味着前沿能力的窗口期越来越短。

对 Daddy 的意义：你选模型时要知道——GLM-5.2 在 sophnet 上的连接不稳定（今天就遇到了），而前沿模型窗口期短意味着选型不能追新，要选稳定可用的。模型经济学也决定了 agent 架构的成本敏感度：每次 subagent 调用都是真金白银。

全阶段：行业认知

⭐ #4 — arXiv：无 Ground-Truth 的强化学习也能提升 LLM

arXiv · Jun 25, 2026

RLVR（Reinforcement Learning with Verifiable Rewards）通常依赖 ground-truth 答案分配奖励。本文探索没有 ground-truth 的情况下用 RL 提升 LLM——这大幅扩展了 RL 可应用的场景（开放式推理、创意写作、agent 策略优化）。

对 Daddy 的意义：你的 loop-engineering "feedback 就是 agent 的训练数据"理念在这里得到了学术支撑——不需要标准答案，只需要verifiable signal。adversarial verifier 的 issues 就是 RLVR 中的 reward signal。未来 agent 可以从 verification feedback 中自我改进。

阶段 1：LLM 基础

⭐ #5 — DSpark：DeepSeek 的推测解码加速 LLM 推理

GitHub/DeepSeek · Jun 27, 2026 · HN 648 pts

DeepSeek 发布 DSpark，一种新的 speculative decoding 方法加速 LLM 推理。Speculative decoding 用小模型快速生成 draft token，大模型批量验证，在不损失质量的情况下显著降低延迟。DeepSeek 持续在推理优化方向输出（DeepSpec 系列）。

对 Daddy 的意义：如果 agent 每次调用要等几秒，multi-step loop 的总延迟会累积。speculative decoding 是降低 agent 延迟的底层手段。结合 06/25 日报的 "Code-as-Plan"（1 次 API 调用替代 80 次），延迟和成本是 agent 架构的两大约束。

阶段 4：推理优化

#6 — arXiv：CHIA — 开源 Agentic AI 硬件/软件协同设计框架

arXiv · Jun 25, 2026

CHIA 是一个开源框架，用 agentic AI 驱动硬件/软件协同设计研究，覆盖计算机架构、系统、编译器和 VLSI。展示了 agent 在专业工程领域的落地——不只是写 CRUD 代码，还能做芯片设计层面的探索。

对 Daddy 的意义：你的 hardware-pm skill 管理电子零件供应商交付——CHIA 的框架设计（principled agentic co-design）可以参考：agent 不是替代人做决策，而是在设计空间中探索 + 验证方案。

💡 核心收获（5 条）

1. Agent 安全防御正在分层化。模型层（训练抵抗 injection）+ 风格层（destyling）+ 架构层（budget cap + stopping condition）。三层叠加才能可靠。

2. Multi-agent 不是银弹，有死循环风险。CVE-2026-LGTM 虽然是 satire，但 340 条评论 + $41K 花费的模式是真实风险。每次 spawn subagent 都要有 budget cap。

3. Feedback = RLVR reward。不需要 ground-truth 答案，只需要 verifiable signal。你的 adversarial verifier 产出的 issues 就是 agent 自我改进的 reward signal。这是 Karpathy "数据→模型→Loss→Backprop" 在 agent 领域的映射。

4. 前沿模型窗口期短，选型追稳定。今天 sophnet/GLM-5.2 断连导致 subagent 全挂——稳定可用的 GLM-5.2 比追新的 GPT-5.6 Sol 更重要。考虑配 fallback provider。

5. Agent 正在进入专业工程领域。CHIA 展示了 agent 在芯片设计中的应用。你的 hardware-pm + mining-assets-mgmt 已经在走这条路——agent 不只是写代码，是在特定领域做探索+验证+执行。
