2. 极简主义的三个原则

原则 1：核心算法极简

去掉所有优化、并行、缓存、分布式——核心还能剩什么？

Agent 的核心 loop 应该能在 ~50 行里描述（plan → execute → observe → verify）。Adversarial Verification 的核心只需要 spawn 两个 agent。

原则 2：从数据中学

microgpt 不需要手工规则来生成名字。它从 32,000 个名字里学。

Agent 从 verification feedback 中学。Closed-loop feedback = agent 的"训练数据"。ComPilot 论文也证实：feedback 闭环比 open-loop 好 23-40%。

原则 3：看原始输出

Karpathy：直接看 loss curve、看生成的样本。不要只看汇总指标。

不要只看 verification_report.json 的 pass/fail。 要看 verifier 的具体 issues。Cognitive surrender 的解药就是"看原始输出"。

3. 从 microgpt 到 agent architecture

Training Loop → Agent Iteration Loop

microgpt training loop:           Agent iteration loop:
─────────────────────────         ─────────────────────
for step in range(num_steps):     for iter in range(max_iterations):
    logits = model(x)                 result = delegate_task(...)
    loss = cross_entropy(logits, y)   issues = verify(result, rubric)
    model.zero_grad()                 feedback = extract_lessons(issues)
    loss.backward()                   strategy.update(feedback)
    optimizer.step()                  if stopping_condition_met(): break
    if step % eval_interval == 0:
        print(loss.item())

同构：forward = 执行，backward = 验证，update = 修正，evaluate = 停止条件。

Inference Loop → Agent Execution Loop

microgpt inference:               Agent execution:
───────────────────               ───────────────
context = [token]                  context = [user_prompt]
while len(output) < max_tokens:    while not done:
    logits = model(context)            plan = think(context)
    probs = softmax(logits[-1])        action = decide(plan)
    next_token = sample(probs)         result = execute(action)
    context.append(next_token)         context.append(result)

两者都是 autoregressive loop——每一步基于累积的 context 决定下一步。

5. 对你工作的启示

5.1 不要过度设计

microgpt 200 行 = 完整 GPT。你的 loop-engineering 三层（core + adversarial + patterns）= ~260 行。刚好 Karpathy 尺度。

5.2 Feedback 就是 agent 的训练数据

每次 verifier 抓到问题 → 一条训练样本。积累 verification_report.json → agent 的"dataset"。未来 agent 可从 feedback 自我改进。

5.3 极简验证原则

看 verifier 写的具体 issues。 一条 issue 比一个 pass/fail 值 100 倍。
